# Apple Silicon Core 待开发清单

面向个人开发，目标是先做出稳定可用的最小版本，再逐步补强。

## 开发原则
- 先稳定，再优化
- 先打通主流程，再扩展兼容性
- 每个阶段都要能独立演示

## P0（必须先完成）

### 1. 设备识别与安全过滤
- [ ] 列出可写入的外置磁盘（名称、容量、设备路径）
- [ ] 默认过滤内置系统盘，禁止误选
- [ ] 标记磁盘挂载状态，写入前统一卸载

验收标准：
- 能稳定展示目标 U 盘，且不会出现内置系统盘可选。

### 2. 镜像基础校验
- [ ] 支持 ISO/IMG 文件路径读取
- [ ] 校验文件存在、可读、大小合法
- [ ] 不支持格式时给出明确错误信息

验收标准：
- 非法镜像能在写入前被拦截，错误信息可读。

### 3. 写入主流程（最小可用）
- [ ] 执行 写入前确认 -> 卸载 -> 写入 -> 结果返回
- [ ] 输出阶段性进度（至少显示当前阶段）
- [ ] 写入失败时返回统一错误码与日志片段

验收标准：
- 可成功把常见 Linux ISO 写入到外置 U 盘。

### 4. 权限与命令执行封装
- [ ] 统一执行提权命令入口
- [ ] 采集 stdout/stderr 与退出码
- [ ] 增加超时与中断处理

验收标准：
- 权限拒绝、命令失败、超时三种场景都有清晰反馈。

## P1（P0 完成后）

### 5. 写入后校验
- [ ] 实现快速抽样校验
- [ ] 提供可选完整校验开关
- [ ] 输出校验报告（成功/失败原因）

### 6. 错误码与日志规范
- [ ] 统一错误码：设备、权限、镜像、写入、校验
- [ ] 统一日志等级：info/warn/error
- [ ] 记录关键耗时，便于后续优化

### 7. 与 common 层边界整理
- [ ] 将可复用逻辑下沉到 mac/common
- [ ] 保持 apple_silicon 与 intel 接口一致
- [ ] 避免目录间循环依赖

## P2（体验与质量提升）

### 8. 稳定性与异常恢复
- [ ] 支持用户取消任务
- [ ] 处理中途拔盘/磁盘被占用等异常
- [ ] 失败后支持安全重试

### 9. 测试补齐
- [ ] 单元测试：设备过滤、镜像校验、错误映射
- [ ] 集成测试：端到端写入流程
- [ ] 异常测试：权限拒绝、空间不足、损坏镜像

## 建议里程碑（个人节奏）

### M1：打通
- 完成 P0，命令行可稳定写入 Linux ISO。

### M2：可诊断
- 完成 P1，出现问题时能快速定位原因。

### M3：可发布内测
- 完成 P2，具备基本恢复能力与测试覆盖。

## 每周执行模板（个人开发建议）
- 周目标：只设 1 个里程碑子目标
- 日推进：每天完成 1 个可验收小项
- 周收尾：整理失败案例与下一周风险
