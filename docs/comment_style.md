# 注释规范（bootoo 项目）

本规范适用于 bootoo 项目所有 Python 代码文件，旨在提升代码可读性、可维护性和团队协作效率。

## 1. 总体原则
- 注释应简洁明了，准确表达代码意图。
- 注释内容应与代码保持同步，代码变更时及时更新注释。
- 禁止无意义、重复代码逻辑的注释。

## 2. 文件头注释
- 每个核心模块建议在文件顶部添加模块说明，包括功能简介、主要数据结构、关键字段说明等。

## 3. 函数/方法注释
- 每个函数/方法应有独立的文档字符串（docstring），说明如下：
  - 功能描述：简要说明该函数的作用。
  - 输入参数：列出所有参数及其类型、含义。
  - 返回值：说明返回类型及结构。
- 示例：

```python
def example_func(a: int, b: str) -> bool:
    """
    功能描述：判断 b 是否为数字且长度等于 a。
    输入参数：
        - a: 期望的长度（int）
        - b: 待判断字符串（str）
    返回值：
        - bool，True 表示条件满足，False 表示不满足
    """
    ...
```

## 4. 行间注释
- 关键处理步骤、易混淆或业务相关的判断应加行间注释。
- 注释应与代码缩进对齐，避免影响代码结构。
- 行间注释应放在代码上方或右侧，内容简明扼要。
- 示例：

```python
for disk in disks:
    # 检查是否为系统盘
    if disk['id'] == 'disk0':
        continue
    ...
```

## 5. 命名与注释风格
- 变量、函数命名应见名知意，减少“此处为何如此”的注释需求。
- 注释统一使用中文，特殊缩写或英文术语应在首次出现时说明。

## 6. 其他建议
- 对于复杂算法、业务流程，可配合流程图、示意图等文档补充说明。
- 对外接口、核心数据结构建议在 README 或专门文档中详细说明。
